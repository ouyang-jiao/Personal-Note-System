# 个人笔记与知识库系统 - 需求文档

## 一、产品概述

个人笔记与知识库系统是一款基于 Vue 3 框架开发的现代化笔记管理应用，旨在帮助用户高效管理个人知识、记录学习笔记、分类整理信息，实现知识的系统化和可视化管理。

## 二、目标用户

- **学生群体**: 需要记录学习笔记、整理课程资料的学生
- **职场人士**: 需要管理工作文档、记录会议纪要的职场人员
- **知识爱好者**: 需要收集整理各类知识的个人用户

## 三、功能需求

### 3.1 用户认证模块

#### 3.1.1 用户注册
- 用户可通过注册页面创建新账户
- 注册信息包括：用户名、密码、确认密码
- 密码长度至少6位
- 两次输入的密码必须一致
- 用户名不可重复

#### 3.1.2 用户登录
- 用户可通过登录页面登录系统
- 登录信息包括：用户名、密码
- 支持"记住我"功能，记住用户名和密码
- 支持下拉选择已记住的用户
- 默认账户：admin / 123456

#### 3.1.3 修改密码
- 登录后可在系统设置中修改密码
- 需要验证原密码
- 新密码长度至少6位
- 两次输入的新密码必须一致

#### 3.1.4 退出登录
- 点击退出按钮可退出当前账户
- 退出后返回登录页面

### 3.2 笔记管理模块

#### 3.2.1 笔记新增
- 标题输入框（必填）
- 内容文本框（必填，支持大文本）
- 分类下拉选择
- 标签多选
- 优先级选择（高、中、低）
- 是否置顶单选框
- 创建时间自动生成
- 保存、清空按钮
- 实时表单校验

#### 3.2.2 笔记编辑
- 支持编辑已有笔记的所有字段
- 保存修改并更新修改时间

#### 3.2.3 笔记查看
- 笔记详情页面展示完整内容
- 展示标题、内容、分类、标签、优先级、创建时间、修改时间

#### 3.2.4 笔记搜索
- 关键词模糊搜索
- 按分类筛选
- 按标签筛选

#### 3.2.5 笔记排序
- 按创建时间排序
- 按修改时间排序
- 按置顶优先排序

#### 3.2.6 笔记操作
- 编辑：修改笔记内容
- 删除：移入回收站
- 置顶：设置笔记置顶
- 取消置顶：取消笔记置顶

### 3.3 分类管理模块

#### 3.3.1 分类列表
- 展示所有分类
- 显示分类名称和笔记数量

#### 3.3.2 分类操作
- 新增分类
- 编辑分类名称
- 删除分类（需确认）
- 分类颜色标识

#### 3.3.3 分类导航
- 侧边栏显示分类列表
- 点击分类筛选对应笔记

### 3.4 标签管理模块

#### 3.4.1 标签列表
- 标签云展示
- 显示标签名称和使用频次

#### 3.4.2 标签操作
- 新增标签
- 删除标签（需确认）

#### 3.4.3 标签筛选
- 点击标签筛选对应笔记

### 3.5 回收站模块

#### 3.5.1 删除笔记列表
- 展示已删除的笔记
- 显示标题、删除时间

#### 3.5.2 回收站操作
- 恢复笔记
- 永久删除笔记（需确认）
- 清空回收站（需确认）

### 3.6 数据看板模块

#### 3.6.1 统计概览
- 笔记总数
- 分类数量
- 标签数量
- 今日新增笔记

#### 3.6.2 分类占比图
- 饼图展示各分类笔记占比

#### 3.6.3 月度趋势图
- 折线图展示月度新增笔记数量

#### 3.6.4 标签频次图
- 柱状图展示标签使用频次

### 3.7 系统设置模块

#### 3.7.1 主题设置
- 浅色模式
- 深色模式

#### 3.7.2 密码修改
- 修改当前用户密码

#### 3.7.3 数据统计
- 显示系统数据统计信息

## 四、技术需求

### 4.1 框架与组件库
- **Vue 3**: 渐进式 JavaScript 框架
- **Vue Router 4**: 路由管理
- **Pinia**: 状态管理
- **Element Plus**: UI 组件库
- **ECharts**: 数据可视化

### 4.2 状态管理
- 使用 Pinia 管理全局状态
- 笔记数据状态（笔记、分类、标签）
- 用户认证状态（当前用户、登录状态）

### 4.3 数据持久化
- 使用 localStorage 存储所有数据
- 笔记数据、分类数据、标签数据、用户数据、设置数据
- 页面加载自动读取本地缓存
- 数据变化时自动同步本地存储

### 4.4 表单验证
- 使用 Element Plus 表单校验组件
- 标题不能为空、长度限制
- 密码长度至少6位、两次输入一致
- 实时校验反馈

### 4.5 路由管理
- 多页面路由配置
- 路由守卫控制访问权限
- 未登录用户自动跳转登录页
- 已登录用户访问登录页自动跳转首页

### 4.6 响应式设计
- 适配不同屏幕尺寸
- 移动端侧边栏可折叠

## 五、非功能需求

### 5.1 性能要求
- 页面加载时间 < 2秒
- 操作响应时间 < 500毫秒

### 5.2 可用性要求
- 清晰的视觉层级
- 直观的操作流程
- 友好的错误提示

### 5.3 安全性要求
- 用户密码本地加密存储
- 路由权限控制
- 防止未授权访问

### 5.4 可维护性要求
- 模块化组件设计
- 清晰的代码结构
- 规范的命名约定

## 八、使用流程

### 8.1 首次使用流程
1. 打开应用 → 显示登录页面
2. 使用默认账户登录（admin / 123456）
3. 进入首页查看知识库总览
4. 创建新笔记

### 8.2 日常使用流程
1. 登录系统
2. 浏览/搜索笔记
3. 创建/编辑笔记
4. 管理分类和标签
5. 查看数据统计
6. 退出系统

## 十、风险评估

### 10.1 技术风险
- localStorage 容量限制（约5MB）
- 浏览器兼容性问题
- 深色模式样式覆盖不全

### 10.2 业务风险
- 用户数据丢失风险（本地存储）
- 用户忘记密码无法找回

### 10.3 应对措施
- 提示用户定期导出数据
- 提供修改密码功能
- 定期备份数据到本地文件
