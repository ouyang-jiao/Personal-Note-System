三、Vue 双向绑定原理

3.1 核心概念

Vue 3 的双向绑定基于 Proxy 代理实现，通过 v-model 指令建立数据与视图的实时同步机制。

数据流向图
用户输入
↓
input 事件
↓
v-model 更新数据
↓
Proxy 触发响应
↓
视图自动更新
3.2 代码实现示例

在笔记编辑表单中，使用 v-model 实现标题和内容的双向绑定：

<template>
<el-form :model="form">
<el-form-item label="标题">
<el-input v-model="form.title" placeholder="请输入标题" />
</el-form-item>
<el-form-item label="内容">
<el-textarea v-model="form.content" placeholder="请输入内容" />
</el-form-item>
</el-form>
</template>

<script setup>
import { ref } from 'vue'

const form = ref({
title: '',
content: ''
})
</script>
原理说明

ref：创建响应式引用，当值变化时触发视图更新。

v-model：语法糖，等价于 :value="form.title" @input="form.title = $event"。

Proxy：Vue 3 使用 Proxy 代理对象属性，实现响应式追踪。

四、状态持久化（localStorage）

4.1 实现方式

使用浏览器的 localStorage API 将数据存储在本地，实现刷新页面后数据不丢失。

4.2 核心代码实现

在 Pinia Store 中封装数据加载和保存逻辑：

const STORAGE_KEY = 'notebook_data'

function loadData() {
try {
const data = localStorage.getItem(STORAGE_KEY)
if (data) {
return JSON.parse(data)
}
} catch {}
return getDefaultData()
}

function saveData(data) {
localStorage.setItem(STORAGE_KEY, JSON.stringify(data))
}
4.3 数据结构
数据类型存储内容用途
notes笔记列表（标题、内容、分类、标签等）笔记管理
categories分类列表（名称、颜色）分类管理
tags标签列表（名称、颜色）标签管理
settings系统设置（主题、分页大小等）系统配置
users用户列表（用户名、密码哈希）用户认证
工作流程

页面加载 → 从 localStorage 读取数据 → 初始化 Pinia 状态

数据修改 → 更新 Pinia 状态 → 自动同步到 localStorage

刷新页面 → 从 localStorage 恢复数据 → 状态不丢失

5.2 路由守卫

使用 beforeEach 全局守卫控制页面访问权限：

router.beforeEach((to, from, next) => {
const authStore = useAuthStore()
authStore.init()

if (to.meta.requiresAuth === false) {
if (authStore.isLoggedIn) {
next('/')
} else {
next()
}
} else {
if (authStore.isLoggedIn) {
next()
} else {
next('/login')
}
}
})
权限控制逻辑

requiresAuth: false：登录/注册页面，已登录用户自动跳转到首页

requiresAuth: true：受保护页面，未登录用户自动跳转到登录页

六、Pinia 状态管理

6.1 核心概念

Pinia 是 Vue 3 官方推荐的状态管理库，采用 Composition API 风格，支持模块化 Store。

Pinia Store 结构
State（状态）
↔
Getters（计算属性）
↔
Actions（方法）
6.2 Store 定义示例

定义笔记数据 Store（note.js）：

import { defineStore } from 'pinia'
import { ref, computed } from 'vue'

export const useNoteStore = defineStore('note', () => {
const data = ref(loadData())

const notes = computed(() => data.value.notes.filter(n => !n.isDeleted))
const topNotes = computed(() => notes.value.filter(n => n.isTop))

function addNote(note) {
const newNote = {
id: Date.now().toString(),
...note,
createdAt: new Date().toISOString()
}
data.value.notes.unshift(newNote)
save()
return newNote
}

return { notes, topNotes, addNote }
})
6.3 Store 使用示例

在组件中使用 Store：

<script setup>
import { useNoteStore } from '../stores/note'

const noteStore = useNoteStore()

function handleAddNote() {
noteStore.addNote({
title: '新笔记',
content: '笔记内容',
category: '学习笔记',
tags: ['重要']
})
}
</script>
Pinia 优势

响应式：状态变化自动触发视图更新

模块化：按功能划分 Store，便于管理

TypeScript：良好的类型支持

DevTools：支持时间旅行调试

十、用户认证系统

10.1 用户注册

注册流程：验证用户名唯一性 → 密码哈希 → 保存到 localStorage

function register(username, password, confirmPassword) {
const existingUser = users.value.find(u => u.username === username)
if (existingUser) {
return { success: false, message: '用户名已存在' }
}

const newUser = {
id: Date.now().toString(),
username,
password: hashPassword(password),
createdAt: new Date().toISOString()
}

users.value.push(newUser)
save()
return { success: true, message: '注册成功' }
}
10.2 用户登录

登录流程：验证用户名密码 → 设置当前用户 → 支持记住密码

function login(username, password, remember = false) {
const user = users.value.find(u => u.username === username)
if (!user || user.password !== hashPassword(password)) {
return { success: false, message: '用户名或密码错误' }
}

currentUser.value = { id: user.id, username: user.username }

if (remember) {
rememberedUsers.value.push({ username, password })
}

save()
return { success: true, message: '登录成功' }
}
10.3 密码哈希

使用简单哈希算法加密密码，保护用户隐私：

function hashPassword(password) {
let hash = 0
for (let i = 0; i < password.length; i++) {
const char = password.charCodeAt(i)
hash = ((hash << 5) - hash) + char
hash = hash & hash
}
return hash.toString(16)
}

十二、项目亮点与技术难点

12.1 技术亮点
  • 前后端一体化：无需后端服务器，所有数据存储在浏览器本地
  • 响应式设计：适配不同屏幕尺寸的设备
  • 深色模式：完整的深色主题支持，保护视力
  • 动画效果：丰富的过渡动画和交互反馈
  • 数据可视化：使用 ECharts 展示笔记统计数据
12.2 技术难点
  • 路由守卫：实现登录状态与页面访问权限的联动
  • 状态同步：确保 Pinia 状态与 localStorage 的实时同步
  • 深色模式样式覆盖：覆盖 Element Plus 组件的默认样式
  • 数据一致性：删除分类/标签时同步更新相关笔记
12.3 解决方案
  • 路由守卫：使用 beforeEach 全局守卫，通过路由元信息判断权限
  • 状态同步：在 Pinia 的 actions 中统一处理 localStorage 读写
  • 样式覆盖：使用 CSS 变量和深度选择器（::v-deep）
  • 数据一致性：删除分类/标签时遍历所有笔记进行更新
